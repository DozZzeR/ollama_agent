## ✅ Completed — TOOL_LOOP Refactor

### What changed
- Replaced PLAN pipeline (planner → stepExecutor → evaluator) with simple TOOL_LOOP
- Model gets tool schemas, decides when to call tools via standard Ollama tool_calls
- Orchestrator executes tools and feeds results back to model
- Two modes: DIRECT (score ≤ 1, no tools) and TOOL_LOOP (score ≥ 2, tools available)
- DB schema updated: agent_runs now tracks triage_score, triage_decision, model_response, tools_called
- Anti-hallucination system prompt: forbids claiming tool results without actual calls
- Language matching: CRITICAL RULE in system prompt for both modes
- Fact validation: blocked patterns for meta-instructions and garbage
- `/tools` command replaces `/plan` — forces TOOL_LOOP for next message
